---

# Purpose

This document defines the official Control Point Library used within the BUXA PDMS™ Quality Standards system.

The Control Point Library establishes every measurable inspection item used throughout the project lifecycle.

A Control Point identifies **what should be inspected**.

It does not define acceptance criteria or inspection methods.

Those requirements are maintained separately within the Inspection Standards documentation.

---

# Vision

Every inspection should be repeatable.

Every quality decision should be measurable.

Every project should be evaluated using the same Control Point system.

The Control Point Library creates one consistent inspection language for every project delivered through BUXA PDMS™.

---

# What is a Control Point?

A Control Point is the smallest measurable inspection unit within BUXA PDMS™.

Each Control Point represents one inspection objective.

Examples include:

- 基层含水率
- 基层平整度
- 地板伸缩缝
- 拼缝质量
- 表面保护

Every inspection consists of multiple Control Points.

Every Quality Gate references one or more Control Points.

# Control Point Numbering

Every Control Point receives one permanent identifier.

Example

```
CP-001

CP-002

CP-003
```

Control Point IDs are permanent.

They should never be reused or renumbered.

---

# Control Point Structure

Every Control Point contains the following information.

- Control Point ID
- 控制点名称
- English Reference
- 所属项目阶段
- 所属 Quality Gate
- 检查类别
- 检查目的
- 对应 Inspection Standard
- 备注

This structure ensures consistency across every project.

# Inspection Categories

Control Points may belong to one or more inspection categories.

Examples include:

- 图纸审核
- 产品检查
- 材料检查
- 基层检查
- 环境检查
- 安装检查
- 成品保护
- 外观检查
- 功能检查
- 交付检查
- 售后检查

Categories help organise inspections without changing the project workflow.

---

# Library Maintenance

The Control Point Library is designed for continuous expansion.

New Control Points may be added when:

- New construction methods are introduced.
- New products are released.
- New quality risks are identified.
- Continuous improvement requires additional inspection items.

Existing Control Point IDs should remain unchanged.

---

# Governance

Every project delivered through BUXA PDMS™ shall use the official Control Point Library.

Inspection items should never be created independently for individual projects.

Any new Control Point requires formal review and approval before being added to the library.

---

# Future Structure

The library will gradually expand to include the complete BUXA PDMS™ Control Point system.

Current implementation target:

- 98+ Control Points

Future versions may include additional Control Points without changing the overall framework.
