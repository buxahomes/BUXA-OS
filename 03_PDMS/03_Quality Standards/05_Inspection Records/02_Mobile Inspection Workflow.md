Status: Approved

---

# Purpose

This document defines the standard mobile inspection workflow used by the BUXA PDMS™ Project Delivery Management System.

The workflow is designed to minimise manual data entry while ensuring complete, traceable and high-quality inspection records.

The Flooring Project Engineer focuses on inspecting the project.

The system manages the inspection process.

---

# Design Philosophy

Inspect.

Capture.

Confirm.

Move Forward.

The mobile inspection workflow should be simple, fast and intuitive.

The Flooring Project Engineer should spend time observing the project rather than completing forms.

---

# Core Principles

## Reality First

The workflow captures actual site conditions.

The system never creates inspection facts.

Only the Flooring Project Engineer records project conditions.

---

## Minimum Manual Input

The system automatically provides:

- Project
- Stage
- Quality Gate
- Control Point
- Inspection Standard
- Acceptance Criteria
- Inspector
- Date
- Time

The Flooring Project Engineer only records:

- What happened
- What was measured
- What was observed

---

## Evidence Before Documentation

Inspection evidence is collected first.

Reports are generated afterwards.

Evidence always takes priority over written descriptions.

---

## Exception-Based Workflow

Normal inspections remain simple.

Additional information is only requested when required.

Examples include:

- Failed inspection
- Missing evidence
- AI risk warning
- Rectification required

---

# Step 01

## Select Project

The Flooring Project Engineer selects the assigned project.

The system automatically loads:

- Project Information
- Current Stage
- Pending Inspections
- Outstanding Rectifications

---

# Step 02

## Select Inspection Item

The system displays:

Current Stage

↓

Control Point

↓

Inspection Standard

↓

Inspection Item

The Flooring Project Engineer selects the required inspection.

---

# Step 03

## Capture Project Facts

Depending on the inspection type, the Flooring Project Engineer may:

Enter a measurement

or

Select Pass / Fail

or

Record an observation

or

Record a voice note

or

Capture a photograph

No unnecessary fields are displayed.

---

# Step 04

## Capture Evidence

Evidence may include:

- Photo
- Video
- Instrument Reading
- Voice Note
- Drawing Markup

Evidence is automatically linked to the Inspection Record.

---

# Step 05

## AI Analysis

Where applicable, the system analyses:

- Images
- Measurements
- Instrument readings

AI may provide:

- Suggested Result
- Risk Warning
- Additional Measurement Recommendation
- Possible Defect Identification

AI recommendations never replace the Flooring Project Engineer's judgement.

---

# Step 06

## FPE Decision

The Flooring Project Engineer reviews:

Inspection Facts

↓

Evidence

↓

AI Recommendation

↓

Final Decision

The Flooring Project Engineer remains responsible for the inspection outcome.

---

# Step 07

## Inspection Record Created

After submission, the system automatically creates:

Inspection Record

↓

Evidence Link

↓

Audit Trail

↓

Quality Gate Progress

↓

Project History

No manual report writing is required.

---

# Step 08

## Exception Workflow

If the inspection passes:

Project continues.

If the inspection fails:

The system automatically creates a Rectification Item.

The Flooring Project Engineer confirms the required corrective action.

---

# AI Assisted Inspection

Artificial Intelligence may:

- Read photographs
- Detect visible defects
- Read instrument values
- Identify potential risks
- Suggest additional inspections
- Generate inspection summaries

Artificial Intelligence shall never:

- Modify inspection facts
- Approve inspections
- Close rectifications
- Approve Quality Gates

---

# Workflow Principles

Every inspection shall:

Capture facts.

Capture evidence.

Generate one Inspection Record.

Support future auditing.

Support AI learning.

Support automatic report generation.

---

# Future Expansion

Future versions may support:

- Offline inspection mode
- QR Code project identification

- Automatic measurement import
- BIM location mapping
- Voice-first inspection
- Wearable device support
- Real-time collaboration

without changing the inspection workflow.
