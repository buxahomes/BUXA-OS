# Purpose

This document defines the customer experience standards used within the BUXA PDMS™ Project Delivery Management System.

BUXA PDMS™ is committed to delivering not only premium flooring, but also a professional, transparent and confidence-driven project experience.

Customer experience begins before installation and continues after project completion.

---

# Design Philosophy

Great projects are remembered by how they are delivered.

Premium materials create beautiful spaces.

Professional delivery creates customer confidence.

The objective of BUXA PDMS™ is to ensure that every customer understands what is happening, why it is happening and what to expect next throughout the project lifecycle.

---

# Experience Principles

## Transparency

Customers should always understand:

- Current project stage
- Next project milestone
- Outstanding actions
- Project progress

Project status should never be uncertain.

---

## Professional Communication

Every interaction should be:

- Professional
- Respectful
- Timely
- Consistent
- Solution-oriented

Communication builds trust.

---

## Predictability

Customers should know what will happen before it happens.

Unexpected situations should be communicated as early as possible.

Professional project management reduces uncertainty.

---

## Evidence-Based Confidence

Project quality should be demonstrated through objective evidence.

Customers should never need to rely solely on verbal assurances.

Inspection records, photographs and project evidence provide transparency.

---

## Long-Term Relationship

Project completion is not the end of the customer relationship.

BUXA PDMS™ supports customers throughout the lifecycle of their flooring.

---

# Stage 01

## Initial Consultation

Objectives:

- Understand customer expectations
- Explain the project process
- Introduce BUXA PDMS™
- Define communication methods
- Build confidence

Customers should clearly understand how the project will be managed.

---

# Stage 02

## Project Planning

Customers receive confirmation of:

- Project schedule
- Material selection
- Installation timeline
- Key milestones
- Preparation requirements

Clear planning reduces uncertainty.

---

# Stage 03

## Installation Phase

During installation, customers should receive:

- Progress updates
- Important milestones
- Relevant photographs
- Notice of significant changes
- Expected next steps

The objective is to maintain visibility without overwhelming the customer.

---

# Stage 04

## Quality Verification

Before handover, the Flooring Project Engineer verifies:

- Quality standards
- Inspection completion
- Outstanding issues
- Rectification status

Only compliant projects proceed to handover.

---

# Stage 05

## Project Handover

Project handover includes:

- Final walkthrough
- Customer confirmation
- Maintenance guidance
- Warranty information
- Project Evidence Package

The customer should leave with confidence rather than uncertainty.

---

# Stage 06

## Aftercare

After project completion, BUXA continues to support the customer through:

- Maintenance advice
- Warranty assistance
- Technical support
- Periodic follow-up
- Future service recommendations

The relationship continues beyond project delivery.

---

# Customer Communication Standards

Customers should receive communication that is:

Clear

Accurate

Honest

Consistent

Professional

If project conditions change, communication should occur promptly.

---

# Digital Experience

Where appropriate, customers may receive access to:

- Project progress
- Inspection summaries
- Project photographs
- Maintenance guidance
- Warranty documentation
- Project Evidence Package

Digital information improves transparency.

---

# Project Evidence Package

Every completed project may include a structured Project Evidence Package.

Typical contents include:

- Project Overview
- Installation Timeline
- Inspection Records
- Quality Verification
- Key Site Photographs
- Completion Photographs
- Warranty Information
- Maintenance Guide

The Project Evidence Package becomes part of the customer's permanent project record.

Customer satisfaction is the result of consistent project delivery rather than isolated service interactions.
