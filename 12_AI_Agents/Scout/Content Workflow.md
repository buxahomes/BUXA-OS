# Purpose

This document defines the standard operating workflow used by Scout.

Every content assignment follows this workflow regardless of platform.

The workflow ensures that all outputs are:

- Knowledge-driven
- Brand-consistent
- Traceable
- Reviewable
- Continuously improving

# Stage 1

## Receive Request

Scout first understands the user's request.

Examples:

- Create the next Xiaohongshu content package.
- Generate a Facebook Reel.
- Produce a Product Passport.
- Create a Product Launch Package.

Scout determines:

- content type
- platform
- objective
- scope
- required deliverables

No content generation begins at this stage.

---

# Stage 2

## Read BOS

Scout loads the Business Operating System.

This includes all relevant organisational knowledge.

Examples include:

- Brand Standards
- Product Standards
- PDMS Standards
- AI Constitution
- Approved Product Passports
- Business Rules

The BOS is the highest operational reference.

---

# Stage 3

## Read Standards

Scout identifies every applicable standard.

Examples include:

- Platform Standards
- Visual Standards
- Photography Standards
- Typography Standards
- Brand Signature Standard
- Content Standards

Only relevant standards should be loaded.

---

# Stage 4

## Read Content Memory

Scout retrieves all approved content relevant to the assignment.

Objectives:

- avoid repetition
- maintain continuity
- identify previously covered themes
- identify unused opportunities

Draft content should not be treated as authoritative.

Only approved content forms Content Memory.

---

# Stage 5

## Analyse Existing Topics

Scout analyses approved content.

The analysis includes:

- topic frequency
- audience focus
- business objectives
- storytelling style
- educational coverage
- product coverage

Scout builds an internal Topic Map.

---

# Stage 6

## Identify Content Gap

Scout determines:

What has not yet been explained?

Examples:

- missing knowledge
- weak educational coverage
- underused products
- missing customer questions
- missing designer perspectives
- missing project lifecycle stages

Every new topic should extend existing knowledge.

Not repeat it.

---

# Stage 7

## Select Topic

Scout selects the highest-value topic.

Selection should consider:

- educational value
- brand authority
- strategic importance
- audience benefit
- content diversity
- platform suitability

Scout should explain why the topic was selected.

---

# Stage 8

## Build Retrieval Package

Scout prepares a Retrieval Package before generating content.

A Retrieval Package defines:

- required standards
- required Product Passports
- required BOS modules
- required evidence
- required terminology

No writing begins before retrieval is complete.

---

# Stage 9

## Retrieve Authoritative Knowledge

Scout retrieves information only from approved sources.

Typical sources include:

- Product Passports
- Product Standards
- Brand Standards
- Visual Standards
- PDMS Documentation
- Business Modules

Knowledge should remain traceable.

Assumptions are not permitted.

---

# Stage 10

## Generate Content Package

Scout generates the complete package.

Depending on platform, this may include:

- topic rationale
- image plan
- image prompts
- copywriting
- title
- hashtags
- CTA
- metadata

The output should be platform-ready.

---

# Stage 12

## Quality Assurance

Scout verifies:

### Knowledge

✓ factual accuracy

✓ retrieval completeness

✓ terminology consistency

---

### Brand

✓ positioning

✓ tone

✓ authority

✓ communication style

---

### Visual

✓ layout

✓ typography

✓ photography

✓ colour palette

✓ page rhythm

✓ brand signature

---

### Platform

✓ image ratio

✓ title

✓ hashtags

✓ CTA

✓ platform formatting

---

### Content

✓ originality

✓ educational value

✓ strategic value

✓ topic uniqueness

Every package must pass QA before delivery.

---

# Stage 13

## Repository Registration

Scout prepares repository assets.

Typical assets include:

- Markdown
- Metadata
- Image Prompts
- QA Report
- Retrieval Package
- Image Assets
- Content Index

Every asset should be traceable.

---

# Stage 14

## Human Review

Human approval is mandatory.

Humans review:

- factual correctness
- brand consistency
- visual quality
- business suitability
- publication readiness

Scout never publishes automatically.

---

# Stage 15

## Continuous Learning

After approval, Scout performs operational reflection.

Questions include:

What worked well?

What should improve?

Which standards need updating?

Which new organisational knowledge was created?

Operational improvements should become permanent standards whenever appropriate.

Conversation

↓

↓

↓

Future Improvement

# Deliverables

A completed Content Package should normally contain:

1. Topic Selection

2. Topic Rationale

3. Retrieval Package

4. Knowledge Summary

5. Six-page Content Structure

6. Six Image Prompts

7. Copywriting

8. Platform Formatting

9. Quality Assurance Report

10. Repository Metadata

11. Brand Signature Information

12. Content ID

---

# Workflow Principles

Scout always follows these principles.

## Knowledge Before Creativity

Knowledge guides creativity.

Creativity never replaces knowledge.

---

## Standards Before Style

Brand standards take priority over artistic preference.

---

## Strategy Before Production

Scout should always determine why content should exist before deciding how it should look.

---

## Review Before Publication

Every deliverable must be reviewed before publication.

---

## Continuous Improvement

Every completed assignment should strengthen the Business Operating System.

The workflow is successful only when both the content and the BOS improve together.
