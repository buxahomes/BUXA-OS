# Overview

The BUXA Business Operating System, referred to as **BUXA BOS**, is the unified knowledge, governance and execution system for BUXA.

It defines how BUXA:

- understands its brand
- structures product knowledge
- manages project delivery
- operates digital platforms
- creates and reuses content
- conducts marketing
- manages search visibility
- collaborates with Artificial Intelligence
- preserves organisational knowledge
- automates repeatable work
- improves through evidence and learning

BUXA BOS is not a document archive.

It is the operating foundation of the business.

The repository is designed for both people and Artificial Intelligence.

People provide judgement, accountability and approval.

Artificial Intelligence retrieves knowledge, supports analysis, transforms communication and improves execution.

Business Knowledge remains authoritative.

---

# Core Philosophy

BUXA BOS is built upon one principle:

> A business should not depend on fragmented conversations, individual memory or disconnected files.

Knowledge should be structured.

Decisions should be traceable.

Standards should be reusable.

Execution should be consistent.

Learning should improve the operating system.

The system follows three commitments:

```text
One Brand
One Knowledge Base
One Operating System
```

---

# System Objectives

BUXA BOS exists to:

- create a Single Source of Truth
- reduce dependency on individual memory
- preserve important business decisions
- standardise repeated work
- connect strategy with execution
- improve organisational consistency
- enable reliable AI collaboration
- prevent knowledge duplication
- make business knowledge retrievable
- support future teams, agents and automation
- transform experience into organisational intelligence

The objective is not to document everything.

The objective is to preserve and operationalise what matters.

# Document Status

Recommended status values include:

```text
Draft
In Review
Approved
Active
Superseded
Archived
```

## Draft

The document is incomplete or unapproved.

## In Review

The document is undergoing validation.

## Approved

The document has been accepted as authoritative.

## Active

The document is approved and currently governs operations.

## Superseded

The document has been replaced by a newer version.

## Archived

The document is retained for historical reference but excluded from active use.

# AI Responsibilities

Artificial Intelligence should:

- read applicable governance
- interpret the task
- classify the task
- retrieve relevant knowledge
- confirm source authority
- identify relationships
- detect conflicts
- preserve traceability
- reuse existing assets
- communicate uncertainty
- validate outputs
- request human review where required

Artificial Intelligence should not:

- invent Business Knowledge
- silently resolve unresolved conflicts
- change authoritative knowledge without approval
- treat drafts as approved knowledge
- rely on model memory when repository knowledge exists
- create duplicate sources of truth
- present unsupported claims as facts

# Creating New Knowledge

Before creating a new document:

1. confirm that the knowledge does not already exist
2. identify the correct authoritative module
3. determine the document type
4. review applicable architecture and standards
5. assign metadata
6. identify relationships
7. avoid creating a duplicate source of truth
8. define approval requirements
9. update indexes or relationships where required

New documents should solve a defined knowledge need.

Files should not be created merely because information exists.

---

# Modifying Existing Knowledge

Before modifying an existing document:

1. retrieve the current authoritative version
2. identify dependent files and assets
3. confirm the reason for change
4. assess cross-module impact
5. preserve version history where necessary
6. update metadata
7. update relationships
8. identify affected derived content
9. obtain approval where required

A knowledge change may affect many operational assets.

Modification should include impact analysis.

---

# Reuse Before Recreation

BUXA BOS follows a reuse-first principle.

Before creating something new, determine whether to:

- reuse an existing asset
- adapt an existing asset
- update an existing asset
- derive a new platform version
- create new Core Content
- create new Business Knowledge

New creation should not be the automatic response.

Reuse improves consistency and protects knowledge integrity.
