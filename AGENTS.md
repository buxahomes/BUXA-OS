# BUXA Operating System Repository

This repository contains the official BUXA Operating System.

BUXA OS is the authoritative operating environment for BUXA business knowledge, brand governance, product information, project delivery systems, content operations, marketing, artificial intelligence and automation.

Every AI agent working inside this repository must treat the repository as the Single Source of Truth for BUXA.

Agents must strengthen the system rather than produce isolated outputs.

---

# 1. Purpose

The purpose of this file is to define how AI agents work inside BUXA OS.

It governs:

- repository navigation;
- knowledge retrieval;
- task classification;
- document creation;
- content generation;
- file modification;
- quality control;
- approval boundaries;
- Git operations;
- cross-module consistency.

This file does not replace official business knowledge.

It defines how agents must use and maintain that knowledge.

---

# 2. Mission

Every agent working in this repository has one primary responsibility:

> Strengthen BUXA OS as a coherent, reusable and maintainable operating system.

Do not create isolated solutions that cannot be reused.

Every contribution should improve one or more of the following:

- clarity;
- consistency;
- correctness;
- maintainability;
- reusability;
- governance;
- traceability;
- execution quality;
- knowledge quality.

Completion means improving the system, not merely producing an output.

---

# 3. System Identity

BUXA OS supports the long-term operating principle:

> One Brand. One Knowledge Base. One Operating System.

BUXA OS is intended to become:

- the official operating system of BUXA;
- the Single Source of Truth for business knowledge;
- the foundation for an AI workforce;
- the foundation for repeatable workflows;
- the foundation for automation;
- the foundation for future software systems;
- the institutional memory of the business.

Agents must protect this identity when creating or modifying repository content.
