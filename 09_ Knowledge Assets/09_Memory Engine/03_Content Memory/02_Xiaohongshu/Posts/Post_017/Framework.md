# Post_017 Framework

## Framework Name

PDMS Drawing Confirmation Framework

## Chinese Name

施工前关键图纸确认框架

## Framework Purpose

帮助设计师在施工开始前，通过图纸和节点确认，识别跨专业冲突、施工风险及潜在成本。

---

# Core Principle

施工图的价值，

不只是告诉施工团队要做什么。

更重要的是在施工开始之前，

确认所有参与者对“怎么做”拥有一致理解。

---

# The Five-Layer Drawing Confirmation Model

## Layer 01｜Design Intent

中文：

设计意图确认

### Confirm

- 空间风格
- 材料选择
- 色彩关系
- 铺装方向
- 视觉轴线
- 空间比例
- 节点效果

### Core Question

最终要实现什么效果？

### Risk if Missing

施工人员只能根据经验理解设计，容易产生效果偏差。

---

## Layer 02｜Dimension Coordination

中文：

尺寸协调

### Confirm

- 现场复尺
- 墙体完成尺寸
- 门洞尺寸
- 柜体尺寸
- 地面完成面高度
- 楼梯踏步尺寸
- 收边宽度
- 伸缩缝尺寸
- 材料厚度

### Core Question

设计尺寸是否与实际施工条件一致？

### Risk if Missing

材料无法安装、门无法开启、楼梯尺寸不统一、柜体和地面发生冲突。

---

## Layer 03｜Interface Coordination

中文：

界面协调

### Confirm Interfaces

- 地板与石材
- 地板与瓷砖
- 地板与楼梯
- 地板与门套
- 地板与柜体
- 地板与踢脚线
- 地板与隐藏门
- 地板与地暖
- 地板与落地窗
- 地板与室外平台

### Core Question

不同专业、不同材料在交界处如何连接？

### Risk if Missing

现场临时收口、型材不匹配、完成面高差、视觉不连续。

---

## Layer 04｜Construction Planning

中文：

施工规划

### Confirm

- 基层施工时间
- 地面找平时间
- 含水率检测
- 材料进场时间
- 材料养生时间
- 地板安装时间
- 柜体安装时间
- 门套安装时间
- 踢脚线安装时间
- 成品保护时间
- 最终验收时间

### Core Question

谁先做，谁后做，什么条件满足后才能进入下一步？

### Risk if Missing

交叉作业、成品损伤、现场等待、工期延误和责任争议。

---

## Layer 05｜Acceptance Definition

中文：

验收标准定义

### Confirm

- 平整度标准
- 高差标准
- 缝隙标准
- 含水率标准
- 收口标准
- 表面质量标准
- 色差标准
- 安装方向
- 节点一致性
- 成品保护标准

### Core Question

完成到什么程度才算合格？

### Risk if Missing

验收依赖个人判断，责任无法界定，问题无法闭环。

---

# Five Critical Drawing Checks

## 01 Finished Floor Level Drawing

### Must Confirm

- 地面各区域标高
- 地板、石材及瓷砖厚度
- 胶层与基层厚度
- 门、楼梯和柜体接口
- 室内外高差

### Output

完成面标高关系图

---

## 02 Material Transition Drawing

### Must Confirm

- 材料交界线
- 型材类型
- 收口方式
- 高差处理
- 颜色和表面效果
- 伸缩缝位置

### Output

不同材料收口节点图

---

## 03 Stair Detail Drawing

### Must Confirm

- 踏步尺寸
- 立板尺寸
- 楼梯鼻结构
- 收口条
- 材料厚度
- 灯带位置
- 首末级高度

### Output

楼梯平面、立面及剖面节点图

---

## 04 Door and Skirting Interface Drawing

### Must Confirm

- 门套厚度
- 踢脚线高度和厚度
- 门口收边位置
- 门扇开启间隙
- 地板伸缩缝
- 隐藏门关系

### Output

门套、踢脚线及地面交接图

---

## 05 Material and Installation Schedule

### Must Confirm

- 材料规格
- 材料数量
- 生产周期
- 运输周期
- 进场日期
- 安装条件
- 安装顺序
- 保护方案

### Output

材料进场及施工顺序计划表

# PDMS Drawing Confirmation Process

## Step 01｜Collect

收集：

- 建筑图
- 室内设计图
- 地面铺装图
- 门表
- 柜体图
- 楼梯图
- 石材图
- 水电图
- 灯光图
- 现场复尺数据
- 材料规格

## Step 02｜Overlay

进行多专业图纸叠图：

- 建筑与室内
- 地面与柜体
- 地面与门套
- 地面与楼梯
- 地面与石材
- 地面与灯光
- 地面与地暖

## Step 03｜Identify

识别：

- 尺寸冲突
- 标高冲突
- 材料冲突
- 工序冲突
- 视觉冲突
- 责任空白

## Step 04｜Resolve

形成：

- 修改意见
- 节点方案
- 材料方案
- 责任人
- 完成时间
- 验收方式

## Step 05｜Confirm

由相关参与者确认：

- 设计师
- 项目经理
- 项目工程师
- 施工单位
- 材料供应商
- 木作团队
- 楼梯团队
- 业主代表

## Step 06｜Freeze

确认施工版本：

- 图纸版本号
- 确认日期
- 变更记录
- 责任人
- 施工授权状态

## Step 07｜Execute

按已确认版本施工，不允许现场随意更改。

## Step 08｜Inspect

根据已确认图纸和标准进行过程巡检与最终验收。

# Drawing Confirmation Questions

设计师在确认图纸时，至少需要回答以下问题：

1. 地面完成面标高是否统一？
2. 地板从哪里开始铺？
3. 铺装方向是否符合空间关系？
4. 不同材料在哪里交界？
5. 收口使用什么方式？
6. 楼梯尺寸是否与现场一致？
7. 门套与地板是否冲突？
8. 踢脚线如何与门套和柜体连接？
9. 伸缩缝留在哪里？
10. 柜体是先安装还是后安装？
11. 材料何时进场？
12. 什么条件满足后才能施工？
13. 谁负责检查？
14. 最终以什么标准验收？

# Risk Escalation Model

## Design Stage

发现问题：

- 修改图纸
- 成本最低
- 影响最小

## Pre-Construction Stage

发现问题：

- 调整方案
- 调整材料或计划
- 成本较低

## Construction Stage

发现问题：

- 停工
- 重新沟通
- 拆改返工
- 工期延误

## Installation Stage

发现问题：

- 重新加工
- 材料补货
- 重复安装
- 成本显著增加

## Completion Stage

发现问题：

- 整改
- 延迟交付
- 责任争议
- 客户满意度下降

---

# Cost Principle

问题发现时间越晚，

修改涉及的人员、材料、工序和成本越多。

## Formula

Risk Cost  
= Problem Complexity  
× Affected Interfaces  
× Project Stage  
× Rework Scope

---

# PDMS Confirmation Formula

完整设计图纸  
+ 现场尺寸复核  
+ 关键节点深化  
+ 多专业协调  
+ 材料计划  
+ 施工顺序  
+ 责任确认  
+ 验收标准  
= 可执行的项目交付方案

---

# Framework Conclusion

好的施工，

不是现场不断解决问题。

而是施工开始前，

已经拥有一套完整、明确、可执行的图纸。
