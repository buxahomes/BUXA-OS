# P5

## Page Role

PDMS Workflow

## Headline

PDMS  
为什么坚持  
施工前交底？

## Supporting Statement

因为交底不是开会，

而是把项目从“模糊状态”

带入“可执行状态”。

## Four Unified Elements

### 统一标准

让所有人目标一致，减少理解偏差。

### 统一责任

明确每个节点的责任人，避免推诿。

### 统一节点

明确关键时间点，确保计划可控。

### 统一验收

明确验收标准，保证交付质量。

## PDMS Briefing Closed Loop

### 01 信息收集

收集项目图纸、材料、节点等信息。

### 02 风险识别

识别关键节点与潜在风险。

### 03 组织交底会

设计师、施工方、供应商共同参与。

### 04 确认与记录

确认关键事项，形成交底纪要。

### 05 执行落地

按确认内容执行，过程可追溯。

### 06 验收与闭环

按标准验收，闭环管理。

## Value Statement

一次高质量的交底，

带来的价值远不止于减少返工。

## Five Outcomes

- 提升沟通效率
- 减少误解
- 降低风险
- 保证质量
- 提升客户满意度

## Key Quote

真正减少返工，

不是现场补救，

而是提前预防。

## Visual Purpose

建立完整的PDMS施工前交底闭环，强化系统管理价值。
