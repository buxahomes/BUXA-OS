# Post_011 Framework

## Framework Name

Uncertainty-to-Standard Framework

中文名称：

不确定性转化为标准框架

## Framework Purpose

把施工现场可能发生的临时判断，提前转化为明确的设计、节点、工艺与验收标准。

---

## Stage 01：识别不确定性

识别施工前尚未明确的问题：

- 从哪里开始铺
- 每块板如何排版
- 门口如何收口
- 楼梯如何连接
- 柜体如何收边
- 哪里设置伸缩缝
- 不同材料如何衔接
- 工序如何安排

## Stage 02：形成明确决定

把模糊经验转化为明确方案：

- 确认起铺点
- 确认铺装方向
- 确认排版逻辑
- 确认节点构造
- 确认材料接口
- 确认留缝要求
- 确认施工顺序

## Stage 03：图纸化

将决定记录为：

- 平面深化图
- 排版图
- 节点详图
- 材料编号图
- 工艺说明
- 尺寸标注
- 验收要求

## Stage 04：标准化

确保所有参与方使用同一套依据：

- 设计师
- 项目经理
- Flooring Project Engineer
- 施工团队
- 木作单位
- 石材单位
- 玻璃单位
- 业主代表

## Stage 05：施工交底

施工前统一说明：

- 做什么
- 怎么做
- 谁负责
- 何时完成
- 如何检查
- 不合格如何处理

## Stage 06：过程检查

按照深化结果检查：

- 是否按方向施工
- 是否按编号使用材料
- 是否按节点处理
- 是否按尺寸预留
- 是否按顺序施工
- 是否产生偏差

## Stage 07：验收闭环

将深化结果转化为验收标准：

- 可以执行
- 可以检查
- 可以验收
- 可以追溯

---

## Seven Uncertainties Addressed in the Post

1. 从哪里开始铺  
2. 每块板怎么排  
3. 哪里需要收口  
4. 楼梯怎么连接  
5. 门口怎么处理  
6. 柜体如何收边  
7. 哪里需要留缝

---

## Without Deep Design

现场临时决定  
→ 做法不统一  
→ 反复讨论  
→ 效果偏差  
→ 返工  
→ 延期  
→ 成本增加

## With PDMS Deep Design

问题提前识别  
→ 方案提前确认  
→ 图纸统一  
→ 标准统一  
→ 现场按图执行  
→ 过程可控  
→ 结果稳定

---

## Core Formula

经验判断  
→ 明确方案  
→ 图纸表达  
→ 标准执行  
→ 过程检查  
→ 结果验收

## Management Principle

让现场少决定，才能让交付更确定。
