# Xiaohongshu Content Memory

## Purpose

This directory stores the complete publishing memory of the official BUXA PDMS™ Xiaohongshu account.

It is the historical memory of the account, not a content strategy document.

Every published post becomes part of the long-term business memory of BUXA OS.

The objective is to allow AI to understand:

- what has already been published;
- which topics have been covered;
- how the content evolved over time;
- which positioning has been established;
- which ideas should not be repeated.

---

# Scope

This directory includes:

- Cover titles
- Carousel structure
- Image generation prompts
- Xiaohongshu captions
- Publishing metadata
- Performance records (when available)
- Lessons learned
- Topic evolution

This directory does NOT contain marketing strategy or brand standards.

Those are defined elsewhere inside the official Knowledge Base.

---

```

README.md

Index.md

Post_001.md

Post_002.md

Post_040.md
```

Each post is stored as an independent memory record.

---

# Source of Truth

Business positioning comes from:

- Brand
- PDMS
- Marketing
- Website
- Content Strategy

This directory only records published content.

If any inconsistency exists between a Xiaohongshu post and the official Knowledge Base,

the Knowledge Base always has higher authority.

---

# AI Responsibilities

Before generating any new Xiaohongshu content, AI should:

1. Read Index.md.
2. Identify related historical posts.
3. Review similar topics.
4. Avoid duplication.
5. Preserve BUXA brand positioning.
6. Generate a new post based on official knowledge.

AI should never generate content without checking previous publications.

---

# Post Numbering

Posts are permanently numbered.

Example:

```
Post_001.md
Post_002.md
...
Post_040.md
```

Numbers are never reused.

Deleted posts remain archived.

---

# Required Metadata

Every post should include:

- Post ID
- Title
- Theme
- Category
- Audience
- Keywords
- Publish Date
- Status

When available:

- Views
- Likes
- Saves
- Comments
- Advertising
- Notes

---

# Memory Principle

This directory preserves publishing memory.

It is not intended to rewrite history.

Published posts remain immutable.

Future improvements are recorded as new posts rather than modifying old ones.

---

# Relationship with Knowledge

Knowledge defines:

"What BUXA believes."

Memory records:

"What BUXA published."

Knowledge creates consistency.

Memory creates continuity.

---

# Future Expansion

This memory will eventually include:

- Publishing performance
- A/B testing
- Content clusters
- Topic distribution
- Audience feedback
- AI topic planning
- Automatic duplicate detection

The long-term goal is to enable AI to plan future Xiaohongshu content based on the complete publishing history rather than isolated prompts.
