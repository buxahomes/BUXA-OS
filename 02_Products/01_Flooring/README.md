# Flooring Product Documentation Standard

Version: 1.0

Status: Approved

Owner: Founder

Review Cycle: Every 12 Months

---

# Purpose

This module defines the documentation standards for all BUXA flooring products.

Each flooring product must have one official product document.

Every product document serves as the single source of truth for product information across the entire BUXA ecosystem.

This standard applies to all current and future flooring products.

---

# Documentation Principles

## One Product One File

Each flooring product has one dedicated document.

Example:

HD01 - Pure Mist Oak.md

The product document becomes the official reference for:

- Website
- Product Catalogue
- AI
- Social Media
- Marketing
- SEO
- Internal Training
- Future Automation

---

## Single Source of Truth

Product information must never be duplicated.

Every platform should reference the official product document.

Updates should always be made in the product document first.

---

## Global Product Identity

Each flooring product has one global identity.

Regional sales models do not create separate product identities.

If regional specifications differ, the product document records those differences while maintaining one global product code.

---

## Content Before Marketing

Marketing content is generated from product documentation.

Marketing should never become the source of product information.

---

# Standard Product Structure

Every flooring product document should contain the following sections.

---

## 1. Product Identity

Defines the official identity of the product.

Includes:

- Product Code
- Official Product Name
- Category
- Collection
- Status

---

## 2. Product Story

Explains the inspiration behind the product.

Includes:

- Design Inspiration
- Colour Story
- Lifestyle Positioning

---

## 3. Design Profile

Defines the overall design language.

Examples:

- Modern Coastal
- Scandinavian
- Architectural Luxury
- Organic Modern
- Contemporary Oak

---

## 4. Visual Characteristics

Defines how the product looks.

Includes:

- Colour Tone
- Grain Pattern
- Texture
- Surface Finish
- Knot Characteristics
- Overall Visual Style

---

## 5. Technical Specifications

Contains all technical information.

Examples:

- Dimensions
- Construction
- Wear Layer
- Surface Finish
- Joint System
- Installation Method
- Compatible Underfloor Heating
- Warranty

---

## 6. Regional Specifications

Records approved specification differences between regions.

Examples:

Australia

China

Future Markets

Only genuine specification differences should be recorded.

The product identity remains unchanged.

---

## 7. Recommended Applications

Defines suitable project types.

Examples:

- Residential
- Luxury Homes
- Apartments
- Coastal Homes
- Display Homes

---

## 8. Design Compatibility

Defines recommended matching styles.

Examples:

- Interior Styles
- Wall Colours
- Cabinet Colours
- Furniture Styles
- Skirting Boards
- Stair Components

---

## 9. Image Library

Maintains approved product images.

Recommended categories:

- Lifestyle
- Close-up
- Texture
- Room Scene
- Installation
- Sample

---

## 10. Marketing Reference

Defines approved marketing language.

Includes:

- Core Selling Points
- Key Messages
- Approved CTA
- Product Highlights

---

## 11. SEO Reference

Defines search optimisation.

Includes:

- Primary Keywords
- Secondary Keywords
- Meta Description
- Search Intent

---

## 12. AI Reference

Provides structured knowledge for AI.

Includes:

- Recommended Product Description
- Recommended Tone
- Avoided Terminology
- Typical Customer Questions
- Related Products

---
